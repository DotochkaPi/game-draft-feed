# Game Draft Studio — Social Feed
## Plan v1.0

---

## Суть проекта

Веб-приложение где пользователи генерируют GDD-драфты и видят что генерируют другие. Лента вдохновляет, лайки показывают что цепляет аудиторию.

---

## Страницы

### Главная — Лента (/index.html)
- Карточки GDD в хронологическом порядке (новые сверху)
- Каждая карточка: название концепта (первая строка), жанр, дата, количество лайков, кнопка «Открыть»
- Кнопка «Создать свой GDD» — ведёт на генератор
- Пагинация или бесконечный скролл (начнём с пагинации)

### Карточка GDD (/draft.html?id=xxx)
- Полный текст GDD
- Кнопка «❤️ Нравится» с счётчиком
- Кнопка «Поделиться» — копирует URL
- Кнопка «Создать похожий» — открывает генератор с подсказкой

### Генератор (/create.html)
- Тот же флоу что в game-draft-studio (идея → GDD → редактирование)
- После финализации кнопка «Опубликовать в ленту»
- Публикация сохраняет GDD в базу и редиректит на страницу драфта

---

## Хранилище данных

Без бэкенда — используем **JSONBin.io** (бесплатный REST API для JSON).
- Бесплатно: 10k запросов/день, 1MB на bin
- Один bin = массив всех драфтов
- Каждый драфт: `{ id, title, content, genre, createdAt, likes, likedBy[] }`

Альтернатива если вырастем: Firebase Realtime DB (бесплатный тир).

---

## Структура данных

```json
{
  "drafts": [
    {
      "id": "uuid-v4",
      "title": "Первые 60 символов концепта",
      "genre": "hypercasual / RPG / strategy / ...",
      "content": "полный текст GDD",
      "createdAt": "2026-03-18T10:00:00Z",
      "likes": 12,
      "likedBy": ["fingerprint1", "fingerprint2"]
    }
  ]
}
```

Идентификация пользователя без регистрации: **browser fingerprint** (комбинация userAgent + screen + timezone). Хранится в localStorage. Один пользователь = один лайк на драфт.

## Технический стек

- Vanilla HTML5 + JS (как в game-draft-studio)
- Groq API — генерация GDD (ключ пользователь вводит сам)
- JSONBin.io — хранение и чтение драфтов
- GitHub Pages — хостинг
